# MCP工具服务实现日志

## 时间：2026-03-23

## 做了什么

### 1. 实现 MCP Server（src/mcp_server/server.py）

基于 MCP SDK（v1.26.0）的 FastMCP，实现 stdio 通信的 Local Server，注册三个工具：

**query_user_credit_data(user_id: int)**
- 调用 `build_features_from_parquet(case_ids=[user_id], split="test")` 读取 parquet 测试集
- 从 1061 列中提取 SFT 训练数据要求的 13 个字段
- 列名映射：`mainoccupationinc_384A` ← `max_mainoccupationinc_384A`，`debtoverdue_47A` ← `max_debtoverdue_47A`（这两个字段在 parquet 中带聚合前缀）
- NaN/Inf → None，float 整数 → int，保证输出与 SFT tool_response 格式严格一致

**predict_risk_score(features: str)**
- 接收 JSON 字符串，解析后调用 `predict_credit_risk(features)`
- 返回 `{risk_score, risk_level, top_factors}`

**search_knowledge_base(query: str)**
- 调用 `load_retriever()` 加载 FAISS 索引 + BGE-M3 embedding（HF_HUB_OFFLINE=1 离线加载）
- retriever 做了全局缓存，首次加载约 14s，后续调用 <0.05s
- 返回 top 3 片段拼接文本

### 2. 函数级测试（src/mcp_server/test_tools.py）

直接调用三个工具函数，验证返回格式正确。

### 3. MCP 协议端到端测试（src/mcp_server/test_e2e.py）

用 MCP SDK 客户端通过 stdio 连接 Server，模拟 Agent 真实调用，5 组共 21 项测试全部通过：

| 测试组 | 内容 | 结果 |
|--------|------|------|
| 组1 | 5 个不同 case_id 数据查询 | 13 字段齐全，NaN 清洗正常，含缺失值（None）和极端逾期（2865天）样本 |
| 组2 | 5 组真实特征风险预测 | score 范围 0.12-0.19，均为低风险，格式正确 |
| 组3 | 4 种不同语义 query 检索 | 返回内容与 query 语义匹配，首次 14s（模型加载），后续 0.03s |
| 组4 | 完整流水线（查数据→预测→检索） | 三步串联 1.29s 完成 |
| 组5 | 边界情况（不存在ID/空query/手工特征） | 错误处理正常，无崩溃 |
