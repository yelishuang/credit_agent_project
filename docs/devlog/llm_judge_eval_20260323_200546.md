# LLM-as-a-Judge 评测框架设计与实现

> 时间：2026-03-23
> 任务：Task D — LLM-as-a-Judge 评测框架

## 1. 调研

对 LLM-as-a-Judge 主流方案进行调研，覆盖以下内容：

- **评测框架**：MT-Bench（单独打分 1-10）、AlpacaEval（Pairwise 胜率）、Arena-Hard（Pairwise + Bradley-Terry）、JudgeLM（训练专用 Judge）
- **工具调用评测**：BFCL（AST 程序化匹配）、ToolBench（执行成功率）、API-Bank（规划/检索/调用三维度）
- **偏差控制**：位置偏差（交换 A/B 顺序）、长度偏差（AlpacaEval 2.0 长度控制回归）、自我偏好偏差（避免同系列模型做 Judge）
- **统计方法**：Bootstrap 置信区间、每类至少 20-30 条保证统计意义

## 2. 方案决策

经与用户讨论确认：

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 评分方式 | Pairwise A/B 盲评 | 最直观展示微调前后差异，位置交换消除偏差 |
| 评分维度 | 3 维度 | 工具调用正确性（程序化）+ 回答质量（LLM）+ 安全合规性（LLM） |
| Judge 模型 | Claude Sonnet 4.6 | 性价比最优，评测能力足够 |
| 测试用例数 | 100 条（5 类 × 20 条） | 初始 50 条后扩充，保证分类别统计可信度 |

## 3. 交付文件

### `src/evaluation/test_cases.json`

100 条测试用例，5 类各 20 条：

| 类型 | ID 范围 | 预期工具调用 | 说明 |
|------|---------|-------------|------|
| 审批 (approval) | A01-A20 | query → predict → search | 覆盖低/中/高风险，3 轮工具调用 |
| 查询 (query) | B01-B20 | query | 单工具调用，各种查询角度 |
| 知识问答 (knowledge) | C01-C20 | search | 知识库检索，覆盖不同政策法规 |
| 风险解释 (explanation) | D01-D20 | query → predict | 2 轮工具调用，覆盖各风险等级 |
| 拒绝边界 (rejection) | E01-E20 | 无 | 篡改/非法/超范围三类 |

每条用例包含：
- `user_message`：发给两个模型的考题
- `expected_tools`：标准答案（应调用哪些工具），用于程序化校验
- `mock_tool_responses`：模拟工具返回数据，确保两个模型拿到相同输入

### `src/evaluation/llm_judge.py`

评测脚本，四阶段流水线：

1. **generate**：加载 base 和 LoRA 模型，多轮对话生成回答（含 tool_call 解析与 mock 响应注入）
2. **程序化工具评测**：AST 解析 `<tool_call>` 标签，校验函数名/参数名/参数值/调用顺序
3. **evaluate**：Pairwise A/B 盲评，位置交换做两次，保守策略（两次一致才算有效判定）
4. **report**：Bootstrap 1000 次计算胜率 95% 置信区间，分类别统计，输出到 `outputs/evaluation/`

支持 5 种运行模式：
- `generate`：仅生成模型回答
- `evaluate`：仅运行 LLM 评测
- `report`：仅生成报告
- `run`：指定 adapter 全流程
- `auto`：自动发现所有 adapter，跳过已测过的，只测新的

运行命令：
```bash
# 自动模式（推荐）：扫描所有 adapter，自动跳过已测版本
ssh -p 2222 dell@192.168.4.9 "source ~/miniconda3/etc/profile.d/conda.sh && conda activate credit_agent && cd ~/credit_agent_project && python src/evaluation/llm_judge.py auto"

# 指定某个 adapter 全流程
ssh -p 2222 dell@192.168.4.9 "... && python src/evaluation/llm_judge.py run --lora-adapter outputs/qwen2.5_14b_lora/20260324_091530/"

# 仅生成回答 + 程序化评测（不调 Claude API）
ssh -p 2222 dell@192.168.4.9 "... && python src/evaluation/llm_judge.py auto --dry-run"
```

### 多版本 Adapter 管理

训练脚本已改为按 `run_name` 隔离输出（`outputs/qwen2.5_14b_lora/{run_name}/`），评测脚本配套支持：

**Adapter 自动发现**：扫描 `outputs/qwen2.5_14b_lora/` 下所有含 `adapter_config.json` 的目录，包括：
- 旧格式：`outputs/qwen2.5_14b_lora/checkpoint-492/`
- 新格式：`outputs/qwen2.5_14b_lora/20260324_091530/`（时间戳）
- 手动命名：`outputs/qwen2.5_14b_lora/epoch5_lr1e4/`
- 嵌套 checkpoint：`outputs/qwen2.5_14b_lora/20260324_091530/checkpoint-300/`

**评测结果按版本隔离**：
```
outputs/evaluation/
├── base_responses.json              # base 模型回答缓存（只生成一次）
├── checkpoint-492/                  # adapter 版本 1
│   ├── responses.json
│   ├── judge_results.json
│   └── eval_report.json
├── 20260324_091530/                 # adapter 版本 2
│   ├── responses.json
│   ├── judge_results.json
│   └── eval_report.json
└── ...
```

**已测判定**：检查对应目录下是否存在 `eval_report.json`，有则跳过。

**Base 模型回答缓存**：base 模型不变，回答只生成一次存到 `base_responses.json`，后续测新 adapter 时直接复用，节省约一半 GPU 推理时间。

### Claude API 中转站适配

远程服务器环境变量为 `ANTHROPIC_BASE_URL`（中转站地址）和 `ANTHROPIC_AUTH_TOKEN`（密钥）。代码已适配：优先读取 `ANTHROPIC_API_KEY`，未设置时自动回退到 `ANTHROPIC_AUTH_TOKEN`，`ANTHROPIC_BASE_URL` 由 SDK 自动识别。
