# CreditAgent：基于大模型微调的消费贷款智能审批系统

## 一句话定位

客户经理输入一个客户 ID，系统自动完成「拉数据 → 风险评分 → 政策合规检查 → 生成审批报告」全流程，将原本 30 分钟的人工审批分析压缩到 10 秒。

## 业务背景

Home Credit 是一家国际消费金融公司，核心客群是征信记录薄弱甚至空白的人群（征信白户）。这类客户在传统银行很难获得贷款，但 Home Credit 通过替代数据和机器学习模型来评估他们的还款能力，提供普惠消费贷款。

传统审批流程中，客户经理需要：
1. 手动登录多个系统拉取客户数据（收入、负债、逾期记录等）
2. 人工计算关键指标（负债收入比、逾期频率等）
3. 翻阅风控手册确认政策要求
4. 撰写审批意见报告

本系统用 AI Agent 自动化这个全流程。

## 各模块的角色

| 模块 | 做什么 | 为什么需要它 |
|------|--------|-------------|
| LightGBM ML 模型 | 输出精确的违约概率（如 0.00414） | LLM 算不准数值，传统 ML 擅长这个 |
| 微调 Qwen2.5-14B | 驱动 Agent 循环：理解意图、调用工具、分析结果、生成报告 | 通用 LLM 不懂信贷术语，微调后专业性提升 |
| RAG 知识库（FAISS + BGE-m3） | 检索消费金融监管规则、信用评分专业知识、审批场景 QA、数据字典等 | 政策会更新，RAG 可随时替换文档，不用重新训练 |
| MCP 工具协议 | 封装数据查询、模型预测为标准化工具接口 | Local(stdio) 处理本地数据查询和模型推理 |

## ML 模型与 LLM 的分工

这是面试核心问题——为什么两个都要？

- ML 模型：擅长数值预测。给定 610 维特征，输出违约概率 0.00414，精度高但不会解释。
- LLM：擅长综合分析。把「违约概率 0.00414 + 关键因素是贷款金额和逾期记录 + 巴塞尔协议要求」串成一份人能读懂的审批报告。

两者各司其职，Agent 把它们串起来。

## 实际输出示例

输入：`predict_credit_risk({'case_id': 57543})`

ML 模型返回：
```json
{
  "risk_score": 0.00414,
  "risk_level": "低风险",
  "top_factors": [
    "平均 - 已关闭信贷合同起始日期（值: -186）",
    "贷款金额/信用额度（值: 2e+04）",
    "最大出生日期（值: -1.48e+04）",
    "已发放贷款金额（值: 2e+04）",
    "Maximum annuity previously obtained（值: 2.81e+05）"
  ]
}
```

Agent 拿到这些原始数据后，结合 RAG 检索到的风控政策，生成结构化的中文审批报告。
